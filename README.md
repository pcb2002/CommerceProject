# Java Console Commerce System

## 프로젝트 소개
본 프로젝트는 자바(Java) 기반의 콘솔 쇼핑몰 결제 시스템입니다. 
상품 조회, 장바구니 담기, 결제 및 재고 처리 등의 기본 커머스 기능(step1 - Lv.1)부터, CRUD 기반의 관리자 모드(Lv.2), 그리고 **Enum, 람다(Lambda), 스트림(Stream API)**을 활용한 고급 데이터 처리(Lv.3)까지 점진적으로 구현되었습니다.

특히 객체 간의 결합도를 낮추고 역할과 책임(SRP)을 명확히 분리하는 객체지향적 리팩터링에 집중하여 설계되었습니다.

## 주요 기능 상세 (Features)

### 1. 장바구니 및 주문 시스템 (Lv. 1)
- **장바구니 관리:** 사용자가 선택한 상품을 장바구니에 담고, 수량과 총 금액을 동적으로 계산합니다.
- **재고 연동 시스템:** 상품을 담을 때 실시간으로 재고를 검증하며, 최종 주문 확정 시 원본 상품의 재고를 차감합니다.
- **결제 UI/UX:** 주문 전 장바구니 내역과 총 결제 금액을 명확하게 콘솔에 출력하고 주문 확정/취소 프로세스를 제공합니다.

### 2. 관리자 모드 및 상품 관리 (Lv. 2)
- **관리자 인증:** 비밀번호 기반의 관리자 인증 시스템을 제공하며, 3회 이상 실패 시 접근을 차단합니다.
- **상품 CRUD 로직:** - 특정 카테고리에 새로운 상품을 추가할 수 있으며, 동일 카테고리 내 중복된 상품명을 철저히 검증합니다.
  - 기존 상품의 가격, 설명, 재고를 선택적으로 수정할 수 있습니다.
  - 상품 삭제 시, 현재 사용자들의 장바구니에 담긴 동일 상품까지 연동하여 안전하게 제거합니다.

### 3. Enum & Stream API를 활용한 데이터 처리 (Lv. 3)
- **고객 등급별 할인 (Enum):** `BRONZE(0%)`, `SILVER(5%)`, `GOLD(10%)`, `PLATINUM(15%)` 등급을 `Enum`으로 정의하고, 할인율 계산 로직을 내부로 캡슐화하여 다형성 기반의 결제 금액 산출을 구현했습니다.
- **조건부 상품 필터링 (Stream):** `100만원 이하 상품 보기` 등의 기능을 `stream().filter()`를 활용해 선언적으로 필터링하여 출력합니다.
- **장바구니 아이템 검색/제거:** 스트림과 람다식을 활용해 장바구니에 담긴 특정 타겟 상품을 효율적으로 찾아내고 제거합니다.

## 설계 및 아키텍처 (Refactoring Highlights)
지속적인 코드 리뷰와 피드백을 반영하여 아래와 같은 구조적 개선을 이루어냈습니다.

1. **중앙 집중형 DB 클래스 구축:** 모든 엔티티 데이터(상품 목록, 주문 내역, 카테고리 등)를 하나의 `Database` 클래스에서 관리하며, 속성을 `private`으로 캡슐화하여 데이터 무결성을 보장합니다.
2. **역할 분담과 수평적 상호작용 (SRP):** 함수의 무분별한 연쇄 호출(Chaining)과 `void` 메서드 남발을 줄이고, 값을 명확히 리턴받아 처리하는 수평적 객체 협력 구조로 개선했습니다.
3. **도메인 분리:** `Commerce System`(상품 조회 및 UI 로직)과 `Order System`(결제 및 장바구니 비즈니스 로직)의 역할을 명확히 나누어, 시스템 간의 불필요한 DB 참조를 최소화했습니다.
4. **하나의 프로세스로 정리된 흐름:** 각 시스템 내에서 파편화된 메서드들을 응집도 높은 하나의 비즈니스 프로세스(메서드)로 통합하여 가독성과 유지보수성을 극대화했습니다.

## 실행 방법
- **Language:** Java (JDK 8+)
- **IDE:** IntelliJ IDEA 권장
- **Run:**
  1. 저장소를 Clone 합니다.
  2. `src` 폴더 내부의 `Main.java`를 실행합니다.
  3. 콘솔 창에 출력되는 번호 메뉴를 통해 커머스 시스템을 테스트합니다.
